### Session 4 — Security Fixes (All CRITICAL + HIGH)

Fixed all critical and high severity vulnerabilities:

#### Fix 1: Open Redirect in Auth Callback
- **File:** `apps/web/src/app/auth/callback/route.ts`
- **Issue:** `next` query parameter accepted arbitrary URLs for redirect
- **Fix:** Added `sanitizeRedirectPath()` — validates path starts with `/`, rejects `//` (protocol-relative), verifies hostname stays on origin via `new URL()`, falls back to `/builder`

#### Fix 2: SSRF in Media Upload
- **File:** `apps/web/src/app/api/media/upload/route.ts`
- **Issue:** Fetched arbitrary user-provided URLs with no validation
- **Fix:** Added `ALLOWED_IMAGE_DOMAINS` whitelist (nanobanana, replicate, openai CDNs), HTTPS-only enforcement, 15-second `AbortController` timeout, 10MB size limit (Content-Length + buffer check), `image/*` content-type validation, sanitized file extensions

#### Fix 3: Webhook Forgery (NanoBanana)
- **File:** `apps/web/src/app/api/webhooks/nanobanana/route.ts`
- **Issue:** No signature verification — anyone could POST fake webhook events
- **Fix:** Added `verifyWebhookSignature()` using `crypto.createHmac('sha256')` with `crypto.timingSafeEqual()`. Reads body as raw text first, verifies, then JSON-parses

#### Fix 4: Guest Cookie Bypass
- **Files:** `lib/guest-token.ts` (new), `app/api/auth/guest/route.ts` (new), `login/page.tsx`, `middleware.ts`
- **Issue:** Plain `spb-guest=1` cookie trivially forgeable by anyone via browser DevTools
- **Fix:** Guest tokens are now server-signed HMAC tokens (`{timestamp}.{hmac}`) created via a POST to `/api/auth/guest`. Middleware verifies the HMAC signature and checks expiry (24h). Cookie is `httpOnly`

#### Fix 5: Hardcoded User IDs
- **Files:** `app/api/media/route.ts`, `app/api/checkout/route.ts`
- **Issue:** Both routes used `"00000000-0000-0000-0000-000000000000"` instead of real auth
- **Fix:** Replaced with `createClient()` + `supabase.auth.getUser()`. Returns 401 if not authenticated. Also removed `error.message` leak from checkout error response

#### Fix 6: Missing Auth on AI Routes
- **Files:** `lib/require-auth.ts` (new), all 5 AI route files
- **Issue:** All 5 routes (polish, refine, styles, suggest, variations) had zero auth — unauthenticated users could burn OpenAI credits
- **Fix:** Created shared `requireAuth()` helper that calls `supabase.auth.getUser()` and returns a typed `{ user }` or `{ error: 401 Response }`. Added to all 5 routes

#### Fix 7: Stripe Webhook Replay + Error Leak
- **File:** `app/api/webhook/stripe/route.ts`
- **Issue:** No idempotency check (replay attacks possible), leaked `err.message` in signature failure response
- **Fix:** Added in-memory `processedEvents` Set with 1000-event cap for deduplication. Removed `err.message` from error response

#### Fix 8: Security Headers
- **File:** `next.config.ts`
- **Issue:** No security headers configured at all
- **Fix:** Added `headers()` function returning: `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Referrer-Policy: strict-origin-when-cross-origin`, `Permissions-Policy: camera=(), microphone=(), geolocation=()`, `X-DNS-Prefetch-Control: on`

#### Fix 9: Input Validation on Image Generation
- **File:** `app/api/generate/route.ts`
- **Issue:** No validation on prompt length, model (no allowlist), count (no bounds), aspectRatio. Also leaked `error.message` and upstream error data
- **Fix:** Added zod schema: `prompt` (1–2000 chars), `model` (enum of 4 allowed values), `count` (1–4 int), `aspectRatio` (enum of 7 ratios), `negativePrompt` (max 1000, optional). Added auth via `requireAuth()`. Removed all error detail leaks

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Server-signed HMAC for guest tokens | Prevents trivial cookie forgery while keeping guest mode functional |
| In-memory Set for Stripe idempotency | Lightweight, sufficient for current scale. Note: resets on redeploy — use DB/Redis for production |
| Shared `requireAuth()` helper | Avoid duplicating auth boilerplate across all API routes |
| Domain whitelist for media upload | More secure than URL regex; easy to extend as new providers are added |
| Removed all `error.message` from responses | Prevents internal error/stack trace leakage to clients |
| Kept rate limiting as future work | Requires infrastructure decision; auth enforcement already blocks the worst abuse |
